Highlights

  1. Task: Continual adaptation of pre-trained robot policies where robots learn new tasks sequentially while retaining previously acquired skills.
  2. Method: Propose RFC (Residual-Field Consistency) as a geometry-aware metric to guide when and how residual policies can be safely merged, overcoming the failure of task-similarity metrics in residual RL.
  3. Performance: Better forward transfer, lower backward transfer (less forgetting), and higher task success rates—achieved with minimal data (no demonstrations) and parameter-efficient merging.

Motivation: When New Tasks Keep Arriving

Consider a scenario: you have a generalist robot policy and a robot deployed in a home or factory environment. The robot needs to continuously learn 5–10 new tasks as they arrive sequentially—each task Tk comes with its dataset Dk, and the policy must be adapted to handle all tasks without degrading performance on previously learned ones.

Challenges

Existing continual learning (CL) methods are not well suited for adapting pre-trained policies:

  1. Poor generalization — IL-based PECFT methods often fail to generalize to new task variants.
  2. Catastrophic forgetting — Sequential fine-tuning causes severe parameter interference and performance collapse on old tasks.
  3. Costly continual data collection — Most PECFT methods rely on large amounts of demonstrations per task.
  4. Parameter inefficiency — Maintaining separate adapters per task leads to linear parameter growth and deployment complexity.

Our Approach

ResMerge addresses these by (1) using residual RL for data-efficient adaptation without demonstrations, and (2) using RFC-guided merging to consolidate compatible residuals, improving parameter efficiency and multi-task capability while avoiding geometric conflicts that cause task-similarity-based merging to fail.

Overview

ResMerge operates in two stages: Learn and Merge. In Learn, a LoRA-based residual policy is trained on each new task via RL while the base policy stays frozen. In Merge, RFC evaluates compatibility between the new residual and existing experts; compatible ones are merged in parameter space, otherwise added as a new expert. This design directly addresses the challenges below.

Experimental Setup

  • Simulation: LIBERO benchmark (LIBERO-Spatial and LIBERO-Object), each with 10 sequential long-horizon manipulation tasks. Franka robot in tabletop kitchen setting. Base policies: 32M ViT-based policy and 8B OpenVLA-OFT.
  • Baselines: Seq. SFT, ER, PackNet, TAIL, DMPEL, Multi-Task.
  • Metrics: Forward Transfer (FWT), Negative Backward Transfer (NBT), AUC, Model Parameter Efficiency (MPE), Data Cost.

Results

Main Results

ResMerge consistently outperforms baselines, demonstrating the generalization and no-forgetting claims in Highlights.

Visualization — RFC vs. Task-Similarity Merge

Task-similarity-based merging often combines residuals with conflicting correction fields, causing performance degradation. RFC evaluates direction and scale consistency of residual corrections, enabling reliable merging and higher post-merge success rates (e.g., Merge-SR 78% vs. 21% for merge-all, 54% for no-merge).

Real-world Deployment

Five sequential manipulation tasks (Pick-Banana, Grab-Cube, Put-Pepper, Arrange-Flower, Insert-Pen) on Franka Emika Panda with Octo-small base policy. ResMerge attains FWT 67%, NBT 10%, AUC 76%, significantly outperforming DMPEL (FWT 24%, NBT 23%, AUC 54%), validating the framework's practical applicability and the Highlights' performance claims.
